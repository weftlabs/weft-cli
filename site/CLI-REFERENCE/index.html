    CLI Reference¶

    Complete reference for all weft commands.

    Global Options¶

    These options work with all commands:

    weft [GLOBAL OPTIONS] COMMAND [COMMAND OPTIONS]
    Option Description
    --config PATH Path to config file (default: .env)
    --verbose, -v Enable verbose output
    --version Show version and exit
    --help Show help message and exit

    Commands Overview¶

    Command Description
    weft init Initialize a new Weft project
    weft up Start the Docker runtime
    weft down Stop the Docker runtime
    weft logs View logs from agent watchers
    weft submit Submit a prompt to an agent
    weft feature Manage features (create, start, list, status, accept, drop)

    weft init¶

    Initialize Weft in the current project directory.

    Synopsis¶

    weft init [OPTIONS]

    Description¶

    Creates necessary configuration files and directories for Weft:

    • .weftrc.yaml - Project configuration
    • .weft/ - Runtime directory structure
    • .weft/docker-compose.yml - Project-specific Docker configuration
    • .weft/prompts/ - Agent prompt specifications (customizable per project)
    • AI history repository (in parent directory by default)

    If project is already initialized, shows a friendly message.

    Options¶

    Option Type Description
    --project-name TEXT String Name of the project
    --project-type CHOICE backend|frontend|fullstack Type of project
    --ai-provider CHOICE claude|ollama|other AI provider to use
    --ai-history-path PATH Path Path to AI history repository
    --model TEXT String Specific AI model (e.g., claude-3-5-sonnet-20241022)

    If options aren't provided, you'll be prompted interactively.

    Examples¶

    # Interactive initialization
weft init

# Non-interactive with all options
weft init \
  --project-name my-app \
  --project-type fullstack \
  --ai-provider claude \
  --ai-history-path ../weft-ai-history \
  --model claude-3-5-sonnet-20241022

    Exit Codes¶

    • 0 - Success
    • 1 - Error (missing dependencies, permission denied, etc.)

    weft up¶

    Start the Docker runtime environment with agent watchers.

    Synopsis¶

    weft up [OPTIONS]

    Description¶

    Starts Docker containers for all enabled agents defined in .weftrc.yaml. Each agent runs as a "watcher" that monitors for new prompts and processes them using the configured AI backend.

    Validates:

    • Docker is installed and daemon is running
    • .weftrc.yaml exists
    • API keys are set (via environment variables)

    Options¶

    Option Type Default Description
    --detach, -d Flag true Run containers in background
    --no-detach, -D Flag - Run containers in foreground (attach to logs)

    Environment Variables¶

    Required:

    • ANTHROPIC_API_KEY or WEFT_ANTHROPIC_API_KEY - Claude API key

    Optional:

    • AI_BACKEND - Override backend (default: from .weftrc.yaml)
    • CLAUDE_MODEL - Override model (default: from .weftrc.yaml)
    • POLL_INTERVAL - Watcher poll interval in seconds (default: 2)

    Examples¶

    # Start in background (default)
weft up

# Start and attach to logs
weft up --no-detach

# Start with custom model
CLAUDE_MODEL=claude-3-opus-20240229 weft up

    Exit Codes¶

    • 0 - Services started successfully
    • 1 - Docker not installed, daemon not running, or startup failed

    See Also¶

    • weft down - Stop the runtime
    • weft logs - View runtime logs

    weft down¶

    Stop the Docker runtime environment.

    Synopsis¶

    weft down [OPTIONS]

    Description¶

    Stops all running watcher containers and optionally removes volumes.

    Options¶

    Option Type Description
    --volumes, -v Flag Remove named volumes

    Examples¶

    # Stop services (keep volumes)
weft down

# Stop and remove volumes
weft down --volumes

    Exit Codes¶

    • 0 - Services stopped successfully
    • 1 - Docker not available or stop failed

    See Also¶

    • weft up - Start the runtime
    • weft logs - View logs before stopping

    weft logs¶

    View logs from agent watcher containers.

    Synopsis¶

    weft logs [AGENT] [OPTIONS]

    Description¶

    Displays logs from Docker containers running agent watchers. Can show logs for a specific agent or all agents.

    Arguments¶

    Argument Description
    AGENT Agent name (optional). One of: meta, architect, openapi, ui, integration, test

    Options¶

    Option Type Default Description
    --follow, -f Flag false Follow log output (stream live)
    --tail, -n NUM Integer 100 Number of lines to show from end

    Examples¶

    # View last 100 lines from meta agent
weft logs meta

# Stream meta agent logs in real-time
weft logs meta --follow

# View last 50 lines from all agents
weft logs --tail 50

# Stream all logs
weft logs --follow

    Exit Codes¶

    • 0 - Logs displayed successfully (or user interrupted with Ctrl+C)
    • 1 - Docker not available or container not found

    Tips¶

    • Use --follow to watch agents process prompts in real-time
    • Press Ctrl+C to stop following logs
    • Check logs if agents aren't processing prompts

    weft submit¶

    Submit a prompt to a specific agent.

    Synopsis¶

    weft submit FEATURE_ID AGENT [PROMPT_TEXT] [OPTIONS]

    Description¶

    Creates a prompt file in the agent's in/ directory. The watcher for that agent will automatically pick it up and process it.

    Note: This command is partially implemented. Current best practice is to manually create prompt files in the AI history repository.

    Arguments¶

    Argument Description
    FEATURE_ID Feature identifier (e.g., user-auth)
    AGENT Agent to submit to: 00-meta, 01-architect, 02-openapi, etc.
    PROMPT_TEXT Optional prompt text (if not provided, will prompt interactively)

    Options¶

    Option Type Description
    --file PATH Path Read prompt from file instead of text

    Examples¶

    # Submit text prompt
weft submit user-auth 00-meta "Create user authentication system"

# Submit from file
weft submit user-auth 01-architect --file meta_output.md

# Interactive prompt (not yet fully implemented)
weft submit user-auth 00-meta

    Current Workaround¶

    Until weft submit is fully implemented, use:

    # Navigate to AI history
cd ../weft-ai-history/FEATURE_ID/AGENT_ID/in/

# Create prompt file
cat > my_prompt.md << 'EOF'
Your prompt content here
EOF

# The watcher will automatically process it

    weft feature¶

    Manage feature lifecycle.

    Synopsis¶

    weft feature SUBCOMMAND [OPTIONS]

    Subcommands¶

    Subcommand Description
    create Create a new feature
    start Start working on a feature
    list List all features
    status Show feature status
    review Review and decide on feature (accept/drop/continue)
    drop Drop/cancel a feature

    weft feature create¶

    Create a new feature.

    Synopsis¶

    weft feature create FEATURE_NAME [OPTIONS]

    Description¶

    Creates a new feature with:

    • State tracking in .weft/features/
    • Directory structure in AI history repository
    • Preparation for agent execution

    Arguments¶

    Argument Description
    FEATURE_NAME Name of the feature (use kebab-case: user-auth)

    Options¶

    Option Type Description
    --description TEXT String Feature description
    --priority CHOICE low|medium|high Feature priority

    Examples¶

    # Simple create
weft feature create user-authentication

# With metadata
weft feature create user-authentication \
  --description "Add JWT-based user authentication" \
  --priority high

    Exit Codes¶

    • 0 - Feature created successfully
    • 1 - Feature already exists or creation failed

    weft feature start¶

    Start working on a feature.

    Synopsis¶

    weft feature start FEATURE_NAME

    Description¶

    Marks a feature as "in progress" and:

    • Sets up feature workspace
    • Creates agent directories in AI history
    • Prepares for first agent (Meta)

    Arguments¶

    Argument Description
    FEATURE_NAME Name of the feature to start

    Examples¶

    weft feature start user-authentication

    Exit Codes¶

    • 0 - Feature started successfully
    • 1 - Feature not found or already started

    weft feature list¶

    List all features.

    Synopsis¶

    weft feature list [OPTIONS]

    Description¶

    Shows all features with their current status.

    Options¶

    Option Type Description
    --status CHOICE pending|in-progress|completed|dropped Filter by status

    Examples¶

    # List all features
weft feature list

# List only in-progress features
weft feature list --status in-progress

    Output Format¶

    FEATURE NAME            STATUS         CREATED
user-authentication     in-progress    2024-01-15 10:30
payment-integration     pending        2024-01-16 14:22
admin-dashboard         completed      2024-01-10 09:15

    weft feature status¶

    Show detailed status of a feature.

    Synopsis¶

    weft feature status FEATURE_NAME

    Description¶

    Displays:

    • Feature metadata
    • Current status
    • Which agents have processed
    • Location of outputs
    • Next steps

    Arguments¶

    Argument Description
    FEATURE_NAME Name of the feature

    Examples¶

    weft feature status user-authentication

    Output Format¶

    Feature: user-authentication
Status: in-progress
Created: 2024-01-15 10:30

Agents:
   00-meta       Completed
   01-architect  Completed
   02-openapi    Pending
  - 03-ui         Not started
  - 04-integration Not started
  - 05-test       Not started

AI History: /path/to/weft-ai-history/user-authentication/

    weft feature review¶

    Review a feature's outputs and decide next steps (accept/drop/continue).

    Synopsis¶

    weft feature review FEATURE_NAME [OPTIONS]

    Description¶

    Presents an interactive review of all agent outputs and allows you to:

    • Accept: Merge feature to main branch and mark as completed
    • Drop: Abandon feature and cleanup (optionally delete AI history)
    • Continue: Keep feature active for further work

    The review displays:

    • Summary of all agent outputs
    • Generated files and changes
    • Test results (if available)
    • Code applied to feature branch

    Arguments¶

    Argument Description
    FEATURE_NAME Name of the feature to review

    Options¶

    Option Description
    --delete-history When dropping, delete AI history instead of archiving
    --reason TEXT Reason for dropping (for audit trail)
    --base-branch TEXT Target branch for merge (default: main)

    Examples¶

    # Interactive review
weft feature review user-authentication

# Review with custom base branch
weft feature review user-authentication --base-branch develop

    Exit Codes¶

    • 0 - Review completed successfully
    • 1 - Feature not found or review cancelled

    weft feature drop¶

    Drop/abandon a feature without merging.

    Synopsis¶

    weft feature drop FEATURE_NAME [OPTIONS]

    Description¶

    Drops a feature and:

    • Removes the worktree and branch
    • Marks AI history as "DROPPED" (preserved by default for audit)
    • Updates feature state
    • Optionally deletes AI history permanently

    By default, AI history is preserved for audit compliance. Use --delete-history to permanently remove it.

    Arguments¶

    Argument Description
    FEATURE_NAME Name of the feature to drop

    Options¶

    Option Short Type Description
    --delete-history Flag Permanently delete AI history (default: preserve for audit)
    --reason -r String Reason for dropping (added to audit trail)
    --force -f Flag Skip confirmation prompt

    Examples¶

    # Drop a feature you created but never started
weft feature drop my-feature

# Drop and delete all AI history
weft feature drop my-feature --delete-history

# Drop with a reason (for audit trail)
weft feature drop my-feature -r "Requirements changed"

# Skip confirmation (be careful!)
weft feature drop my-feature --force

# Combine options
weft feature drop my-feature --delete-history -r "Test cleanup" -f

    Exit Codes¶

    • 0 - Feature dropped successfully
    • 1 - Feature not found or drop failed

    Environment Variables¶

    Weft respects these environment variables:

    Variable Description Example
    ANTHROPIC_API_KEY Claude API key (required for AI) sk-ant-api03-...
    WEFT_ANTHROPIC_API_KEY Alternative name for API key sk-ant-api03-...
    AI_BACKEND Override AI backend anthropic, local
    CLAUDE_MODEL Override Claude model claude-3-5-sonnet-20241022
    POLL_INTERVAL Watcher poll interval (seconds) 2
    CODE_REPO_PATH Override code repository path /path/to/repo
    AI_HISTORY_PATH Override AI history path /path/to/history

    Exit Codes¶

    All Weft commands follow standard exit code conventions:

    Code Meaning
    0 Success
    1 General error
    2 Misuse (invalid arguments)

    Shell Completion¶

    Weft supports shell completion for Bash, Zsh, and Fish.

    Bash¶

    # Add to ~/.bashrc
eval "$(_WEFT_COMPLETE=bash_source weft)"

    Zsh¶

    # Add to ~/.zshrc
eval "$(_WEFT_COMPLETE=zsh_source weft)"

    Fish¶

    # Add to ~/.config/fish/completions/weft.fish
eval (env _WEFT_COMPLETE=fish_source weft)

    Configuration Files¶

    Weft looks for configuration in this order:

    1. Command-line --config option
    2. .env in current directory
    3. ~/.weft.env in home directory
    4. Environment variables
